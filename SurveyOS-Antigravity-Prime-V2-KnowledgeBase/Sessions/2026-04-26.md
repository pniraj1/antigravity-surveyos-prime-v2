## What We Did

### 1. Valuation / Break-in Inspection Report (NEW feature)

New standalone `SurveyType: 'valuation'` for pre-insurance inspections when there is a break in the insurance period. Distinct from Spot/Final claim workflow — no accident, no assessment rows.

**Flow:** New Claim → Details → Documents (RC auto-fill) → Valuation (condition form) → Photos → Report

- `'valuation'` added to `SurveyType` union in `src/types/report.ts`
- `ValuationConditionRow` + `ValuationDetails` interfaces in `src/types/assessment.ts`
- `valuationDetails` field + blank default added to `ClaimData` / `createBlankClaim` in `src/types/claim.ts`
- `updateValuationDetails()` action added to `src/stores/claim-store.ts` (immutable spread, same pattern as `updateSpotDetails`)
- `'valuation'` added to `AppTab` union in `src/stores/ui-store.ts`
- Sidebar: new `CarFront` nav item; valuation survey hides `assessment`, `fees`, `reinspection`, `review` tabs
- Dashboard: dynamic import + `TabPlaceholder` entry for `ValuationTab`
- `NewClaimDialog`: third button "Valuation" with break-in hint text
- `DetailsTab`: fixed stale `'reinspection'` comparison in `surveyLabel` (TypeScript caught it)

### 2. New Files Created

> [!tip] Key Decisions
> 1. **New SurveyType, not just ReportType** — gives valuation its own tab set and workflow isolation
> 2. **Panel damage rows inside ValuationDetails** — reuses `SpotDamageRow` shape, no separate store actions needed
> 3. **HTML inline preview only** — no PDFViewer for valuation (simpler, faster render, consistent with spot)
> 4. **NVIDIA NIM for vault logs** — free model, zero Claude tokens burned on session logging
